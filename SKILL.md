# Site Text Audit

## Назначение
Проверяет сайт на орфографические, грамматические, логические и стилистические ошибки.

## Когда использовать
Если пользователь просит:
- проверить сайт на ошибки
- сделать аудит сайта
- проверить орфографию сайта
- проверить грамматику сайта
- найти ошибки на сайте

## Команда запуска

Всегда запускай через exec:

/home/qifa/openclaw-env/bin/python /home/qifa/.openclaw/workspace/skills/site_text_audit/task.py PAGES

Где:
- PAGES — количество страниц для проверки

Важно:
- URL не передаётся
- сайт всегда фиксированный: https://www.qifa.ru

## Пример
Пользователь:
проверь сайт https://www.qifa.ru/ на 5 страниц

Команда:
python3 ~/.openclaw/workspace/skills/site_text_audit/task.py "https://www.qifa.ru/" 5

## Поведение

Если пользователь просит:
- проверить сайт
- аудит сайта
- найти ошибки

→ всегда запускай task.py

Если пользователь не указал количество страниц:
→ сначала спроси, сколько страниц проверять

НЕ анализируй сайт вручную
НЕ придумывай ответ

##Ответ пользователю и Как передать файл пользователю

Никогда не используй команды `exec: mkdir`, `cp`, `mv`. Вместо этого всегда применяй инструмент **write**.

Порядок действий:
1.  Создай итоговый файл **сразу** в папке `/home/qifa/.openclaw/canvas/`.
    Для этого используй write с абсолютным путём:
    write( file_path="/home/qifa/.openclaw/canvas/report.xlsx", content=... )
2.  Явно укажи пользователю ссылку, подставив внешний IP-адрес (если не задан — выполни `exec: curl -s ifconfig.me`).
    Пример ответа: "Файл готов: http://IP_АДРЕС:8888/report.xlsx".

## Правило отправки уведомлений

Ты должен ВСЕГДА и ОБЯЗАТЕЛЬНО уведомлять меня о завершении работы.
Как только твоя задача (анализ, создание файла, поиск) будет полностью выполнена, отправь уведомление с помощью команды:
`openclaw gateway wake --text "✅ Задача выполнена: [краткое описание результата]. Файл: [ссылка на файл]" --mode now`
Не заставляй меня ждать и проверять. Отправляй уведомление сам, как外卖-доставщик сообщает о прибытии заказа.
